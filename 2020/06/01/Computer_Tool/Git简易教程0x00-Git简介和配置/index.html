Git简易教程0x00-Git简介和配置

Posted on 2020-06-01 | In Computer Tool

简单介绍Git与初步配置

什么是Git

Git是一个免费开源的分布式版本控制系统，被用来快速有效的处理各种从小到大项目。

版本控制

版本控制就是记录文件内容的变化以便查阅文件各种版本修改情况的系统。当你目前的项目出现了某个问题，你可以查看项目不同版本的状态，查看是谁对项目进行怎样地修改，并将项目回退到某个时间点之前的状态。使用版本控制比起手动保存项目的各个版本要更加方便，更加有效。

本地版本控制

最简单的方式就是将整个项目复制来保存不同的版本。这种方法极容易出现错误，比如不小心把两个项目的文件混淆，就可能导致出现错误的文件并覆盖了原文件。

为了解决这一系列的问题，人们开发了本地版本控制系统，大多数的本地版本控制系统使用数据库来记录不同版本的文件之间的差异。

集中版本控制

本地版本控制已经能够解决如何进行本地版本控制的问题，但是当在不同系统上的开发者进行团队工作时就需要新的版本控制系统。集中式版本控制就是将版本控制系统放在一台服务器上，这个服务器的版本控制系统集中保存和管理所有文件的版本，所有的开发者都通过客户端连接到这台服务器查看和提交更新。

这种方法的优点是只需对一台服务器进行版本控制，管理比较方便。缺点是当这台服务器出现故障时，所有的开发者都无法进行工作。

分布式版本控制

分布式版本控制中客户端会将整个代码仓库进行完整的镜像，每个开发者都可以先在自己的客户端进行单独的开发，再将修改进行提交到版本控制系统中，其他的开发者也可以从版本控制系统得到新的版本。

这种方法中，不同的开发者都有完整并相对独立的仓库，并且与其他开发者的仓库进行交互。优点是冗余性高，安全性和便捷性非常高。缺点是每个开发者之间的仓库具有多个版本，版本之间管理比较复杂。

Git初次配置

配置设置和查看

在初次使用Git的时候，需要使用git config命令来进行配置。Git的配置具有三种不同等级的作用域，仓库的配置会使用最近作用域的配置，使·开发者可以灵活进行设置。

  • git config --system会对该系统的所有用户进行配置，该系统的所有仓库使用这一配置。因为是系统配置，所以可能需要管理员权限来进行修改。
  • git config --global会对当前用户进行配置，该用户的所有仓库都使用这一配置。
  • git config --local会对当前的仓库进行配置，只有该仓库使用这一配置。

使用git config --list --show-origin命令可以查看Git的所有配置及这些配置所在的作用域。

用户信息

首先配置用户信息以便于在和其他开发者进行协同开发的时候表明自己的身份。

设置文本编辑器

设置默认编辑器，Git会默认调用该编辑器，若没有进行配置，就会使用操作系统默认的文本编辑器。这里默认使用VIM作为默认编辑器。

格式转换

Windows和Linux之间的换行符不同，所以不同系统之间的开发者需要使用格式转换功能保证格式的统一。Git可以自动进行不同平台间的格式转换。

设置为true会在检入时将Windows风格换成Linux风格，检出时将Linux风格换成Windows风格。

设置为input会在检入时将Windows风格换成Linux风格，检出时不进行转换。

设置为false不进行自动转换功能。
